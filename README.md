# MAG CAMP — المرحلة الأولى

نظام Flask لإدارة سكن ولي العهد، جاهز للرفع إلى GitHub ثم النشر على Render.

## محتويات المرحلة الأولى
- تسجيل الدخول بالرقم الوظيفي.
- صلاحيات مدير الخدمات، مدير السكن، 11 مشرف سكن، 3 مراقبي سكن، مدير الصيانة، ومشرف الصيانة.
- إجبار المستخدم على تغيير كلمة المرور الافتراضية عند أول دخول.
- عرض العمال والغرف والبحث فيها.
- كل مشرف يرى نطاق غرفه فقط.
- مدير السكن ومدير الخدمات يريان جميع البيانات والمستخدمين.
- قاعدة البيانات تحتوي على 4,347 عاملًا و968 غرفة.

## بيانات الدخول الأولى
- مدير السكن: `109753`
- كلمة المرور الافتراضية لجميع الحسابات: `123456`

## الرفع إلى GitHub
ارفع جميع الملفات والمجلدات الموجودة بجوار `app.py` إلى جذر المستودع كما هي، بما فيها:
`requirements.txt` و`render.yaml` و`data` و`static` و`Procfile`.

## النشر على Render
1. أنشئ Blueprint جديدًا من مستودع GitHub.
2. سيقرأ Render ملف `render.yaml` تلقائيًا.
3. أمر البناء: `pip install -r requirements.txt`
4. أمر التشغيل: `gunicorn --workers 2 --threads 4 --timeout 120 app:app`
5. فحص التشغيل: `/health`

> ملاحظة: قاعدة SQLite المرفقة مناسبة للمرحلة الأولى والتجربة. في مرحلة التشغيل الدائم مع تحديثات كثيرة، يُفضّل الانتقال إلى PostgreSQL أو إضافة قرص دائم على Render.
